# Governance

XAI Explorer follows the governance structure of the main XAI project.

## Scope

This repository governs:
- Block explorer frontend (React/Vite)
- Explorer backend API
- Transaction and block visualization
- Address and validator dashboards

## Decision Process

### Code Changes

1. **Proposal**: Author creates GitHub issue or PR
2. **Review**: Core team and community provide feedback
3. **Approval**: At least one core team approval required
4. **Merge**: Approved changes merged to main

### Feature Requests

1. **Issue**: Author creates feature request issue
2. **Discussion**: Community feedback period
3. **Prioritization**: Core team evaluates for roadmap
4. **Implementation**: Approved features are scheduled

## Roles

### Core Maintainers
- Review and merge PRs
- Coordinate releases
- Triage issues
- Define roadmap priorities

### Contributors
- Submit PRs following CONTRIBUTING.md
- Participate in discussions and code reviews
- Report bugs and suggest improvements

## Becoming a Maintainer

1. Demonstrate sustained quality contributions
2. Show understanding of the codebase and architecture
3. Receive nomination from core team
4. Approval via consensus of current maintainers
